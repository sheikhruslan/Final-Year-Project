# Final Year Project: Insurance Fraud Detection (Frontend Snapshot)

## What this project is
This repository contains a frontend-focused snapshot of my Final Year Project on insurance fraud detection.

The original project explored how machine learning, statistical checks, and explainability can support fraud investigation workflows. For portfolio and review purposes, this version is intentionally trimmed to the presentation-facing UI layer.

## Why I built it
Insurance fraud is costly, and manual review teams are often overloaded. The project goal was to make investigations faster and more consistent by giving reviewers a clearer way to:
- understand suspicious patterns,
- compare model behavior,
- and make decisions with evidence, not just a single score.

## What’s included here
Current source structure:
- `frontend/src/components/Navbar.tsx`
- `frontend/src/pages/ResearchPaper.tsx`
- `frontend/src/index.css`

This version is mainly the research/reporting interface, designed to communicate:
- project context,
- methodology,
- comparative results,
- and decision-oriented interpretation.

## Highlights recruiters may care about
- **Product thinking:** The interface is written as if it were used by real stakeholders, not only for classroom demo.
- **Communication focus:** The page is structured to make technical work understandable to non-technical reviewers.
- **Data storytelling:** Emphasis on turning model outcomes into practical decisions.
- **UX cleanup and mobile considerations:** Navigation and layout were simplified for readability and small-screen use.

## Tech used in the full project
In the full FYP build (beyond this trimmed snapshot), I worked with:
- React + TypeScript (frontend)
- FastAPI/Python services (backend)
- ML/statistical fraud-detection workflow concepts
